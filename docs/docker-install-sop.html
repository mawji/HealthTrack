Install Guide

Set up your private HealthTrack

A single command puts your personal health dashboard + AI coach on a machine you control, reachable securely from your phone and computer over an encrypted private network. Your data stays on your box — nothing is hosted by anyone else.

About 20–30 minutes, most of it one-time account setup. You'll copy/paste a few commands into a terminal — no coding.

What you'll have at the end

  • HealthTrack running on your own machine, at a private web address like https://healthtrack.your-tailnet.ts.net.
  • The app installed like a native app on your phone and computer (PWA).
  • Your health data synced from Google Health, an AI coach on the model you choose, and one-click updates — all private to you.

Before you start — what you'll need

  • A machine that stays on: a small cloud VPS (≈$5–10/month) or an always-on home PC/Mini-PC. Linux is easiest; 2 GB RAM is fine (4 GB if you want the local AI option).
  • A free Tailscale account (this is your private network + automatic HTTPS).
  • The Tailscale app installed on every device you'll use HealthTrack from — your phone, your laptop. (Free, from the app stores / tailscale.com/download.)
  • A Google account (to connect your Google Health data).
  • Optional: a ChatGPT subscription or an AI API key for the best coach (you can also run a free local model).
No spare machine? A cloud VPS is the simplest path — it's always on, reachable anywhere, and the setup below is identical. Any provider's smallest Linux instance works.

1 · The machine & Docker

1

Get a Linux box and install Docker

On a fresh Ubuntu VPS (or a Linux PC), install Docker with one command:

curl -fsSL https://get.docker.com | sh

On Windows/Mac, install Docker Desktop instead. That's the only software you need on the box — the app itself is delivered as a ready-built image, nothing to compile.

2 · Tailscale — your private network & HTTPS

Tailscale is what lets you reach the app securely from anywhere with a real, browser-trusted HTTPS address — no ports opened to the internet, no certificates to fiddle with.

2

Create the account & turn on HTTPS

  1. Sign up at tailscale.com (free).
  2. In the admin console → DNS → HTTPS Certificates → Enable. (This is required and off by default — without it the app can't get its HTTPS certificate.)
  3. Go to Settings → Keys → Generate auth key. Tick Reusable, copy the key — you'll paste it into the installer.
3

Install the Tailscale app on your devices

Install Tailscale and sign in on each device you'll use HealthTrack from — your phone and your computer (app stores, or tailscale.com/download). This is what lets those devices reach your private .ts.net address. One-time per device.

3 · Google Health connection

4

Create a Google OAuth client

This lets HealthTrack read your health data. In the Google Cloud Console:

  1. Create a project, then enable the "Google Health API".
  2. OAuth consent screen → add your own Google address as a Test user.
  3. Credentials → Create OAuth client ID → "Web application". Save the Client ID and Client secret — the installer will ask for them.

You'll add the exact redirect address in step 5, once you know your .ts.net URL.

4 · Run the installer

5

One command

On your machine, run:

curl -fsSL https://raw.githubusercontent.com/mawji/HealthTrack/main/deploy/install.sh | bash

It asks for your Google client ID/secret, your timezone, and your Tailscale auth key, then brings everything up and prints your app's URL (like https://healthtrack.your-tailnet.ts.net). It figures out the web address for you automatically.

Prefer to look before it starts? Add --no-start to write the settings file and stop. Want the bundled local AI too? Add --local-ai.

5 · Finish in the browser

6

Connect everything

  1. Register the redirect URL with Google. Back in Google Cloud → your OAuth client → Authorized redirect URIs → add, exactly: 
    https://healthtrack.your-tailnet.ts.net/api/googlehealth/callback
    (use your real URL from step 4).
  2. Open your app URL → Settings → connect Google Health.
  3. Settings → AI Provider → pick your coach: connect ChatGPT (Subscription) with a one-time code (no API key), paste an API key for any provider, or use the local model. A cloud model is recommended for the best coaching.
  4. Optional: connect Telegram for reminders and chat-to-log.

6 · Install the app on your phone & computer

Make sure Tailscale is running and signed in on that device (step 3) — the app's address only works on your private network.

Phone: open your app URL in the browser. Android/Chrome: menu → "Install app". iPhone/Safari: Share → "Add to Home Screen". It opens full-screen like a normal app and stays signed in.

Computer: open the URL in Chrome or Edge → click the install icon in the address bar. It gets its own window and icon.

Keeping it up to date

When a new version is available, HealthTrack shows an "Update available" banner in Settings. Click Update now — it backs up your data, updates, and restarts (a few seconds). That's it.

Prefer the command line? cd into your install folder and run docker compose pull && docker compose up -d.

FAQ & quick fixes

QuestionAnswer
The page won't load on my phoneCheck the Tailscale app is on and signed in on that phone. The .ts.net address only works on your private network.
"Your account does not support getting TLS certs"Enable HTTPS Certificates in the Tailscale admin console (step 2), then run docker compose restart in your install folder.
Google sign-in failsMake sure the redirect URL you added in Google (step 5) matches your app URL exactly, including https:// and /api/googlehealth/callback.
Change a setting laterMost settings are in the app's Settings page. The few in the .env file: edit it, then run docker compose up -d in your install folder.
Where's my data?In a data folder next to your install — all on your machine. Back that folder up and you've backed up everything.
Is my data private?Yes — it lives only on your machine. If you choose a cloud AI model, your coaching messages go to that provider; pick the local model for fully on-device AI.
